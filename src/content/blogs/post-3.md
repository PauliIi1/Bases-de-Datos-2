## 1. Bloques Anónimos

### 1.1 Primo o No Primo

Determina si el día actual del mes es un número primo, usando una subconsulta que cuenta los divisores del número.

```sql
-- AUTOR: Paula Buitrago Velandia
-- FECHA: 2026
-- DESCRIPCIÓN: Verifica si el día de hoy (DD) es primo.
--              Un número primo tiene exactamente 2 divisores: 1 y él mismo.

DECLARE
    vd_fecha TIMESTAMP := SYSTIMESTAMP;  -- vd = variable de tipo date/timestamp
    vn_primo NUMBER;                     -- vn = variable numérica

BEGIN
    -- Contamos divisores del día actual usando CONNECT BY LEVEL
    SELECT COUNT(*) INTO vn_primo
    FROM dual
    WHERE (
        SELECT COUNT(*)
        FROM (SELECT LEVEL i FROM dual CONNECT BY LEVEL <= TO_CHAR(SYSDATE, 'DD'))
        WHERE MOD(TO_CHAR(SYSDATE, 'DD'), i) = 0
    ) = 2;  -- Si tiene exactamente 2 divisores, es primo

    -- Evaluamos el resultado
    IF vn_primo = 1 THEN
        dbms_output.put_line('Es primo');
    ELSE
        dbms_output.put_line('No es primo');
    END IF;
END;
```

**Conceptos clave:**
- `CONNECT BY LEVEL`: genera filas del 1 al día actual para iterar sobre posibles divisores.
- `MOD(n, i) = 0`: verifica si `i` es divisor de `n`.
- `COUNT(*) = 2`: un primo solo tiene exactamente 2 divisores.

### 1.2 Serie de Fibonacci

Imprime la secuencia de Fibonacci desde 0 hasta que el siguiente número supere 100.

```sql
-- AUTOR: Paula Buitrago Velandia
-- FECHA: 2026
-- DESCRIPCIÓN: Genera la serie de Fibonacci mientras el valor sea <= 100.
--              Usa tres variables que rotan en cada iteración del ciclo.

DECLARE
    vn_num1 NUMBER := 0;  -- vn = variable numérica | Primer número de la serie
    vn_num2 NUMBER := 1;  -- Segundo número de la serie
    vn_suma NUMBER := 0;  -- Acumulador de la suma

BEGIN
    -- Imprimimos el primer valor antes del ciclo
    dbms_output.put_line(vn_num1);

    -- Ciclo: continúa mientras el número actual no supere 100
    WHILE vn_num2 <= 100 LOOP
        dbms_output.put_line(vn_num2);

        -- Rotamos los valores: suma → num2 → num1
        vn_suma := vn_num1 + vn_num2;
        vn_num1 := vn_num2;
        vn_num2 := vn_suma;
    END LOOP;
END;
```

**Conceptos clave:**
- `WHILE ... LOOP`: repite mientras la condición sea verdadera.
- Rotación de variables: la clave es guardar la suma antes de sobrescribir `num1`.

## 2. Funciones

### 2.1 Saludo con Función

Función que recibe un nombre como parámetro y retorna un saludo personalizado.

```sql
-- AUTOR: Paula Buitrago Velandia
-- FECHA: 2026
-- DESCRIPCIÓN: Función reutilizable que concatena "hola " con el nombre recibido.
--              Luego se llama desde un bloque anónimo para probar su uso.

-- Parte 1: Crear la función
CREATE OR REPLACE FUNCTION fn_saludo_varchar (param_nombres VARCHAR2)
RETURN VARCHAR2
IS
    vv_nombre VARCHAR2(50);  -- vv = variable de tipo varchar
BEGIN
    -- Concatenamos el saludo con el nombre recibido
    vv_nombre := 'hola ' || param_nombres;
    RETURN vv_nombre;
END;
/

-- Parte 2: Llamar la función desde un bloque anónimo
DECLARE
    nombre VARCHAR2(50);
BEGIN
    -- Asignamos el resultado de la función a la variable
    nombre := fn_saludo_varchar('Paula');
    dbms_output.put_line(nombre);  -- Salida: hola Paula
END;
/
```

**Conceptos clave:**
- `CREATE OR REPLACE FUNCTION`: crea o reemplaza la función sin necesidad de borrarla primero.
- `RETURN VARCHAR2`: define el tipo de dato que retorna la función.
- `||`: operador de concatenación en PL/SQL.

### 2.2 Integral Numérica de x² (Método de Riemann)

Función que aproxima la integral definida de x² entre dos límites usando sumas de Riemann.

```sql
-- AUTOR: Paula Buitrago Velandia
-- FECHA: 2026
-- DESCRIPCIÓN: Calcula la integral de x² entre un límite inferior y superior,
--              dividiendo el intervalo en N subdivisiones (método de Riemann derecho).
--              Con 10,000 subdivisiones el resultado se acerca a 1/3 ≈ 0.3333.

-- Parte 1: Crear la función
CREATE OR REPLACE FUNCTION fn_integral_x2 (
    param_Linferior     NUMBER,   -- Límite inferior del intervalo
    param_Lsuperior     NUMBER,   -- Límite superior del intervalo
    param_subdivisiones NUMBER    -- Número de rectángulos para aproximar
)
RETURN NUMBER
IS
    vv_dx   NUMBER;        -- vv = variable numérica | Ancho de cada rectángulo
    vv_suma NUMBER := 0;   -- Acumulador del área total
    vv_x    NUMBER;        -- Punto x evaluado en cada iteración

BEGIN
    -- Calculamos el ancho de cada subdivisión
    vv_dx := (param_Lsuperior - param_Linferior) / param_subdivisiones;

    -- Sumamos el área de cada rectángulo: f(x) * dx = x² * dx
    FOR i IN 1..param_subdivisiones LOOP
        vv_x    := param_Linferior + i * vv_dx;       -- Punto derecho del rectángulo
        vv_suma := vv_suma + (POWER(vv_x, 2) * vv_dx); -- Área del rectángulo
    END LOOP;

    RETURN(vv_suma);
END;
/

-- Parte 2: Llamar la función
DECLARE
    resultado NUMBER;
BEGIN
    -- Integral de x² de 0 a 1 con 10,000 divisiones
    resultado := fn_integral_x2(0, 1, 10000);
    dbms_output.put_line('Resultado: ' || resultado);  -- ≈ 0.3333
END;
/
```

**Conceptos clave:**
- `FOR i IN 1..n LOOP`: itera exactamente N veces.
- `POWER(x, 2)`: eleva x al cuadrado, equivale a x².
- **Método de Riemann:** el área bajo la curva se aproxima como la suma de rectángulos delgados. A más subdivisiones, más preciso el resultado.

## 3. Cursores

### 3.1 Cursor de Empleados — Departamento 80

Cursor explícito que consulta nombre y apellido de empleados del departamento 80.

```sql
-- AUTOR: Andres Gonzalez (adaptado)
-- FECHA: 02/03/2026
-- DESCRIPCIÓN: Cursor explícito que obtiene el primer registro de empleados
--              del departamento 80 y lo imprime en consola.
--              Convención de nombres: primera letra = c (cursor) / v (variable),
--              segunda letra = tipo de dato (v = varchar, n = number, d = date).

DECLARE
    -- Definición del cursor con su consulta asociada
    CURSOR cursor_empleados
    IS
        SELECT FIRST_NAME, LAST_NAME, DEPARTMENT_ID
        FROM tduartec.employees
        WHERE DEPARTMENT_ID = 80;

    -- Variables que recibirán los datos del cursor
    vv_F VARCHAR2(100);  -- vv = variable varchar | Nombre (first name)
    vv_L VARCHAR2(100);  -- vv = variable varchar | Apellido (last name)
    vv_D NUMBER(9);      -- vn = variable numérica | ID del departamento

BEGIN
    -- 1. Abrimos el cursor (ejecuta la consulta)
    OPEN cursor_empleados;

    -- 2. Traemos el primer registro al bloque
    FETCH cursor_empleados INTO vv_L, vv_F, vv_D;

    -- 3. Mostramos el resultado
    dbms_output.put_line(vv_L || ' ' || vv_F || ' ' || vv_D);

    -- 4. Cerramos el cursor para liberar recursos
    CLOSE cursor_empleados;
END;
/
```

**Conceptos clave:**
- **Cursor explícito:** el programador controla manualmente `OPEN`, `FETCH` y `CLOSE`.
- `FETCH ... INTO`: transfiere una fila del cursor a las variables declaradas.
- Convención de variables: `vv_` para VARCHAR2, `vn_` para NUMBER, `vd_` para DATE/TIMESTAMP.
